## 事情的起因

那是一个普通的周三深夜。

我已经写完了功能，跑通了测试，甚至连 README 都更新了——这在我的职业生涯里属于极小概率事件，应当被载入史册。

我端起咖啡，准备享受一个胜利者的睡前仪式。就在这时，控制台亮起了一行红字：

```
TypeError: Cannot read properties of undefined (reading 'length')
```

我微笑了一下。

「小东西。」我想。

## 第一阶段：傲慢（00:00 - 00:40）

我笃定这是后端的锅。数据没返回，接口挂了，同事的 PR 一定偷偷合并了什么见不得人的东西。

我打开 Network 面板，数据在那儿，整整齐齐，一行不少。

我又打开 Vue DevTools，数据也在那儿，安安静静躺在 `data` 里，像一只无辜的猫。

「奇怪。」我说。

这一句「奇怪」，是所有灾难的序章。

## 第二阶段：怀疑（00:40 - 01:30）

我开始怀疑一切。

怀疑浏览器。于是我清了缓存、换了无痕模式、甚至重启了电脑——不是为了解决问题，是为了给自己一点「我已经尽力了」的心理安慰。

怀疑宇宙。我把变量名从 `list` 改成 `dataList`，又改成 `theListThatShouldHaveLength`。没有用。当然没有用，但这让我感觉自己很忙。

怀疑人生。我打开招聘网站，搜索「转行送外卖 月入过万」，认真看了三分钟，然后默默关掉。

## 第三阶段：求神（01:30 - 02:20）

我打开了 Stack Overflow。

在搜索框里，我郑重地输入：

```
Cannot read properties of undefined javascript please help me i am dying
```

没有结果。当然没有，因为我连问题描述都写不利索。

我转向 AI 助手。它温柔地建议我「检查变量是否为 `undefined`」。

我点开它的建议，又关上。这种「正确的废话」让我感到一种被命运凝视的羞辱。

我又去摸了摸桌上的橡皮鸭，对它讲了一遍我的代码。它沉默地盯着我，眼神里有一种「就这？」的意味。

## 第四阶段：顿悟（02:20 - 02:58）

凌晨两点五十八分。

我盯着屏幕，眼神已经涣散到能看见祖先。

我的目光无意识地滑过那一行：

```js
const list = getData();
console.log(list.length);
```

我突然发现，`getData` 后面，少了点东西。

我加上了一个分号。

不，等等——这不是分号的故事。这是一个我忘记写 `await` 的故事。`getData()` 返回的是一个 Promise，Promise 没有 `length`，所以是 `undefined`。

`await`。

一个词。六个字母。三个小时。

## 终章：与羞愧共处

我默默加上了 `await`。控制台一片洁白，没有任何报错。

我没有欢呼。我没有庆祝。我只是慢慢地合上电脑，像合上一本不愿再翻开的书。

那一刻我明白了一件事：**程序员的自尊，不是被 bug 击碎的，是被自己击碎的。** 而 bug，只是那个站在旁边、递刀子的好心人。

## 后记

第二天我给同事讲了这个故事，全程保持着一种「我已经和命运和解」的平静。

同事听完，沉默了三秒，说：

> 「你那个函数我昨天改成了同步的，不需要 `await` 了。」

我看着她。

她看着手机。

我又看了一眼代码。她说得对，函数确实是同步的。

那昨晚的 bug 到底是哪儿来的？

我至今不知道。

也许它从来就没存在过。也许它只是想来看看我。也许，每个深夜的程序员，都值得拥有一个属于自己的、来无影去无踪的 bug——

让你在三个小时后，学会一件事：

**先睡觉。明天再说。**
